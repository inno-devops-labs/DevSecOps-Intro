Executive Summary

A comprehensive security assessment was conducted against OWASP Juice Shop (v19.0.0) using five scanning tools: ZAP (DAST), Semgrep (SAST), Trivy (container image scanning), Nuclei (misconfiguration scanning), and Grype (SCA). A total of 457 findings were identified across all scanners.

The assessment revealed 24 Critical and 211 High severity findings requiring immediate attention. The majority of vulnerabilities stem from outdated third-party dependencies with known CVEs (Trivy: 194, Grype: 167). Application-level issues include SQL injection, XSS, prototype pollution, and path traversal (Semgrep: 39). Infrastructure misconfigurations such as missing security headers, exposed endpoints, and CORS misconfiguration were also identified (ZAP: 9, Nuclei: 10).

Key Observations

  • Dependency risk dominates: Over 75% of findings come from known CVEs in third-party packages (npm, OS packages). This is typical for Node.js applications with large dependency trees.
  • Cross-tool correlation: Trivy and Grype independently flagged many of the same CVEs, confirming scanner accuracy. Some overlap exists (e.g., CVE-2024-21538 in cross-spawn, CVE-2024-37890 in ws).
  • Application-level risks are exploitable: Semgrep identified 39 findings including SQL injection via Sequelize, eval() with user input, hardcoded JWT secrets, and prototype pollution — these are directly exploitable in Juice Shop.
  • Missing security headers: ZAP and Nuclei both confirmed absence of CSP, HSTS, and Permissions-Policy headers. CORS is set to wildcard (*).
  • Exposed sensitive paths: /ftp/ directory listing, /.git/ exposure, /encryptionkeys, /metrics, and /api-docs/ are all publicly accessible without authentication.

Recommendations

  1. Immediate (0–7 days): Address the 24 Critical CVEs by updating the base image and pinning patched dependency versions. Fix SQL injection and eval() code injection vulnerabilities.
  2. Short-term (7–30 days): Rotate the hardcoded JWT secret. Block access to /.git/, /encryptionkeys, /metrics, and /ftp/ paths. Implement CSP and HSTS headers.
  3. Medium-term (30–90 days): Systematically update all npm dependencies flagged by Trivy/Grype. Remediate prototype pollution and ReDoS patterns. Restrict CORS policy.
  4. Ongoing: Integrate SAST/DAST/SCA scanning into CI/CD pipelines. Configure DefectDojo SLAs and track closure rates over sprint cycles.
