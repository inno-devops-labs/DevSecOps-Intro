Key Risks

  • SQL Injection (CWE-89) — detected by multiple tools (Semgrep, Nuclei). Direct string concatenation into queries in product search endpoint.
  • Critical dependency vulnerabilities — @babel/traverse (CVE-2023-45133, CVSS 9.3) and sequelize (CVE-2023-49085, CVSS 9.8) are exploitable with known PoCs.
  • JWT None Algorithm (CWE-347) — application accepts unsigned JWT tokens, allowing full authentication bypass.
  • Missing security headers — CSP, X-Content-Type-Options absent across multiple pages.
  • Hardcoded JWT secret (CWE-798) — secret exposed in source, all tokens can be forged offline.

Recommended Next Steps

  1. Upgrade @babel/traverse to 7.23.2+ and sequelize to 6.35.1+ immediately (critical severity).
  2. Replace raw SQL string concatenation with parameterized queries / ORM methods.
  3. Remove JWT none-algorithm support and rotate the hardcoded secret via environment variable.
  4. Add CSP and other missing security headers through Express Helmet middleware.
  5. Schedule follow-up scan after fixes to verify resolution.
