# Lab 9 --- DevSecOps: Falco & Conftest

## Task 1 --- Falco Runtime Security

### What was done

-   Deployed Falco with custom rules
-   Generated security events using falcosecurity/event-generator
-   Verified alerts in Falco logs

### Key detections

-   Terminal shell in container
-   Write to /usr/local/bin
-   SSH on non-standard port
-   Read sensitive files (/etc/shadow)
-   Netcat execution (RCE pattern)
-   Fileless execution (memfd)
-   Container escape attempts (release_agent)
Evidence 1 — Baseline alert: Terminal shell in container

Falco detected an interactive shell started inside the test container:

```text
2026-04-02T08:42:26.585506776+0000: Notice A shell was spawned in a container with an attached terminal
rule="Terminal shell in container"
container_name="lab9-helper"
process="sh"
command="sh -lc echo hello-from-shell"
```

**Explanation:** This alert shows that Falco detected shell execution inside a running container. Interactive shells in containers are often suspicious because they may indicate manual debugging, unauthorized access, or post-exploitation activity.

### Evidence 2 — Custom alert: Write Binary Under UsrLocalBin

A custom Falco rule was created to detect writes under `/usr/local/bin` inside a container. The rule triggered successfully:

```text
2026-04-02T08:48:37.833640517+0000: Warning Falco Custom: File write in /usr/local/bin
rule="Write Binary Under UsrLocalBin"
container_name="lab9-helper"
file="/usr/local/bin/pwned.txt"
user="root"
```

A second write was also detected:

```text
2026-04-02T08:48:42.350724561+0000: Warning Falco Custom: File write in /usr/local/bin
rule="Write Binary Under UsrLocalBin"
container_name="lab9-helper"
file="/usr/local/bin/pwned2.txt"
user="root"
```

**Explanation:** This confirms that the custom rule worked correctly. Writing files under `/usr/local/bin` may indicate container drift, tampering, or an attempt to place a malicious executable into a common binary path.

### Evidence 3 — Baseline alert: Fileless execution via memfd_create

Falco also detected a critical fileless execution event generated by the Falco event generator:

```text
2026-04-02T08:49:59.819055763+0000: Critical Fileless execution via memfd_create
rule="Fileless execution via memfd_create"
container_name="eventgen"
proc_exepath="memfd:program"
priority="Critical"
```

**Explanation:** This is a high-severity detection because it indicates execution directly from memory without a normal file on disk. Fileless execution is associated with advanced evasion techniques and is a strong indicator of malicious behavior.

### Result

Falco successfully detected:
- a default runtime shell alert,
- a custom file-write rule,
- and a critical advanced execution technique.

This demonstrates that Falco can provide both baseline runtime protection and custom policy-driven detection for container environments.

## Task 2 --- Conftest (Policy as Code)

### Tools

-   Conftest
-   Rego policies

### Policies enforced

-   No :latest tag
-   runAsNonRoot = true
-   allowPrivilegeEscalation = false
-   readOnlyRootFilesystem = true
-   Drop ALL capabilities
-   CPU/Memory requests & limits required
-   Probes recommended
