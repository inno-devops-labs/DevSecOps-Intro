### Why the difference?

1. `GHSA-23c5-xmqv-rm74` affecting `minimatch 3.0.5` was reported by Grype but not Trivy. The likely cause is a difference in advisory database coverage or the way each tool maps GitHub Security Advisories to npm package versions.

2. `CVE-2015-9235` affecting `jsonwebtoken 0.1.0` and `0.4.0` was reported as Critical by Trivy but not under that CVE identifier by Grype. Grype reports related npm findings primarily under GHSA identifiers, so advisory alias handling and package matching can make the same underlying issue appear different between tools.

### When would you pick each?

Syft and Grype are preferable when the SBOM must be retained as a reusable artifact or signed as an attestation. Their decoupled model allows the same inventory to be rescanned against updated vulnerability databases without pulling or rebuilding the original image.

Trivy is preferable for a simple CI step where one tool should scan containers directly. Its broader scanner set also supports vulnerabilities, secrets, IaC, and configuration checks without separately composing several tools.
