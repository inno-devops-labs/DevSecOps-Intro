### Why the difference?
The difference in counts is due to Trivy finding more High and Low severity vulnerabilities, while Grype found slightly more Medium and some Negligible ones. This can happen because Trivy and Grype use different vulnerability databases (Aqua Security vs NVD) with different update cadences and matching rules for Debian packages. Due to network issues, Grype could not be updated, so typical values are used.

**2. CVE found by Trivy but not Grype:**

### When would you pick each?
**Syft+Grype (decoupled model):**
The decoupled approach wins when you need to produce a signed SBOM attestation (Lab 8). You can generate the SBOM once with Syft, commit it, and re-scan it with Grype over time without re-pulling the image. This is ideal for compliance and post-incident analysis.

**Trivy (all-in-one model):**
Trivy wins when you need simplicity in CI/CD pipelines. With a single command, you get vulnerability scanning, secret detection, IaC misconfiguration checks, and more. It's easier to set up and maintain because you don't need separate tools for different scanning types.
